# TODO List Web Server

Веб-приложение для управления списком задач. Проект предоставляет HTTP API и готовый фронтенд из директории `web`: можно создавать, редактировать, удалять и отмечать задачи выполненными, а также работать с повторяющимися задачами.

## Что делает проект

- хранит задачи в SQLite;
- поддерживает задачи с комментарием и датой выполнения;
- поддерживает повторение задач по правилам:
  - `d N` — раз в N дней;
  - `y` — раз в год;
  - `w 1,2,3` — по дням недели;
  - `m ...` — по дням месяца;
- поддерживает поиск задач;
- поддерживает авторизацию по паролю.

## Выполненные задания со звездочкой

В проекте выполнены все задания повышенной трудности:

- переменные окружения;
- поиск задач;
- авторизация;
- дополнительные правила повторения задач (`w`, `m`);
- запуск проекта через Docker.

## Запуск локально

### Требования

- Go `1.25.3`;
- свободный порт `7540` или другой порт, который вы укажете через переменную окружения;
- путь к файлу базы данных в переменной `TODO_DBFILE`.

### Переменные окружения

Проект использует следующие переменные:

- `TODO_DBFILE` — обязательная переменная, путь к SQLite-базе;
- `TODO_PORT` — порт запуска сервера;
- `TODO_PASSWORD` — пароль для авторизации. Если оставить пустым, авторизация не требуется.

Пример для локального запуска:

```bash
export TODO_DBFILE=./scheduler.db
export TODO_PORT=:7540
export TODO_PASSWORD=1234
go run main.go
```

После запуска приложение будет доступно в браузере по адресу:

```text
http://localhost:<порт, который был указан>/
```

Если задан `TODO_PASSWORD`, сначала откроется страница входа:

```text
http://localhost:7540/login.html
```

## Запуск тестов

Перед запуском тестов нужно:

1. Запустить сервер отдельно.
2. Убедиться, что он использует те же значения порта, базы и пароля, что и тесты.

Параметры в файле [tests/settings.go], которые следует использовать:

- `Port = 7540`
- `DBFile = "../scheduler.db"`
- `FullNextDate = true`
- `Search = true`
- `Token = "..."` — JWT-токен для тестов авторизации

Пример запуска сервера перед тестами:

```bash
export TODO_DBFILE=./scheduler.db
export TODO_PORT=:7540
export TODO_PASSWORD=123
go run main.go
```

После этого в другом терминале:

```bash
go test ./tests/
```

## Запуск через Docker

В проекте есть `Dockerfile`.

### Сборка образа

```bash
docker build -t todo-image .
```

### Запуск контейнера

```bash
docker run -d \
  --name todo-app \
  -p 7540:7540 \
  -v "$(pwd)/scheduler.db:/database/scheduler.db" \
  -e TODO_PORT=":7540" \
  -e TODO_PASSWORD="secret" \
  todo-image
```

Контейнер использует:

- `TODO_PORT=:7540`
- `TODO_DBFILE=/database/scheduler.db`

После запуска приложение будет доступно по адресу:

```text
http://localhost:7540/
```

## Структура проекта

- `main.go` — точка входа;
- `pkg/server` — запуск HTTP-сервера;
- `pkg/api` — HTTP-обработчики и авторизация;
- `pkg/service` — логика расчета дат и правил повторения;
- `pkg/db` — работа с SQLite;
- `web` — фронтенд;
- `tests` — интеграционные тесты.
